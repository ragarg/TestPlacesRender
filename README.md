## Пояснение

1. Был выбран MaxRects Best Short Side Fit алгоритм для упаковки. Классический алгоритм для упаковки текстур в атласы, ситуация похожая, поэтому выбар пал именно на него.
2. Анимация камеры - просто полет двух камер, с переключением на другую в конце анимации. Из минусов, из-за смены перспективы в конце есть визуальный скачок, но посчитал простительным для быстрой реализации анимации. В иделае, надо плавно переключаться с одной камеры на другую в полете, с плавным изменением перспективы. Но так как это достаточно трудоемко, не реалищовывал данных подход тут.

## Архитектура и оптимизация

1. Сейчас используется CSS2DObject. Для лучшей работы при больших колличесвах лучше использовать Troika Text, реализованный с помощью SDF. Такой текст рендериться с помощью WebGL, что дает прирост в происзодительности. Так же истек срок патента на технологию Slug, которая дает возможность на gpu рисовать текст с помощью Безье кривых.
2. В своей реализации я использую 1 геометрию для всех объектов. Но это больше оптимизирует работу CPU и памяти. Для оптимизации работы GPU можно использовать инстанциирование, что не помешает работе рейкаста, так как Three.js возвращает нромер инстанса.
3. Для поиска по ID можно использовать классический Map<id, object>, что даст поиск O(1). Для поиска по координатам, можно использовать различные структыру разбиения пространства. Классическое BVH дерево дает O(log N).
